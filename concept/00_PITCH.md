# 00_PITCH — якір концепту (читати ПЕРШИМ, інквізитор звіряє щосесії)

## ONE-LINER
Синтетичні трейдери для дослідження попиту і тренування політик маркетингу
через адаптивні рекомендації AI-Адвайзера.

## ПІТЧ (30 секунд)
WhiteBIT заробляє на комісії. Юзер тримає стратегію або тихо йде (silent churn) —
втрата обороту без сигналу. Перевіряти на живих дорого й повільно.
Ми будуємо синтетичних трейдерів (персони), що проживають РЕАЛЬНІ бэктест-криві
рекомендованих стратегій. Розрив «обіцяне↔пережите» рухає їхню поведінку (тримати/вийти).
Полігон дає: (1) розмічені дані попиту/відтоку ДО живих юзерів; (2) тренування політик
маркетингу — як AI-Адвайзер має радити, щоб максимізувати утриманий оборот → revshare.
Демо для WhiteBIT (Dmitriy) = доказ що це працює на синтетиці, далі перенос на живих.

## ФОКУС (не втрачати)
Ми робимо МАРКЕТИНГОВИЙ інструмент: що і як радити, щоб юзер лишився й торгував.
НЕ психологічну модель довіри. НЕ науку про trust. trust — робоча фіча, не предмет.

## РАЦІОНАЛЬНИЙ ЯКІР: bandit, НЕ POMDP (рішення Парето)

Інквізитор: якщо сесія повзе до belief/POMDP/observation model/ідентифікованості —
ОПУСКАЙ НА ЗЕМЛЮ цим ланцюгом:

1. Ціль 2 = максимізувати Σ(trust→оборот→revshare) вибором налаштувань Адвайзера.
2. Reward (утриманий оборот) — СПОСТЕРЕЖУВАНИЙ прямо з дій. Не схований.
3. trust-компоненти (satisfaction/dependency/momentum) = ФУНКЦІЇ наблюдаемої історії дій
   (sliding window по логу). Обчислювані як фічі. НЕ сховане стан.
4. Якщо trust = фічі з наблюдаемого → belief НЕ потрібен → observation model НЕ потрібна
   → ідентифікованість НЕ потрібна → POMDP НЕ потрібен.
5. Отже: contextual bandit з історичними фічами. Дешево, вимірно, без ризику недоозначеності.

POMDP потрібен ТІЛЬКИ якщо два однакові наблюдаемі контексти вимагають різних дій через
різну СХОВАНУ історію. Але історія наблюдаема (лог дій) → робиться фічею → bandit вистачає.

Асиметрія ціни помилки:
- bandit замість POMDP → політика субоптимальна, але ПРАЦЮЄ і вимірна.
- POMDP замість bandit → тижні на belief, ризик що не зійдеться, ціль 2 повисає.
→ Обираємо bandit. POMDP — research-трек (як SSR), повернутись лише якщо bandit вимірно впреться в стелю.

## НАСЛІДОК для коду
- TASK 05 / харнес: будувати trust як ФІЧІ з наблюдаемої історії, НЕ observation model.
- Політики: contextual bandit. Не POMDP-solver.
- Знімає проблему: ідентифікованість belief, недоозначеність 3 компонент, кількість епізодів
  «для відновлення схованого стану». Епізоди потрібні для НАВЧАННЯ bandit, не для відновлення belief.
