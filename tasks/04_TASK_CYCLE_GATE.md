# TASK 04 — Gate інструмента: чи цикл дає змістовну trust-динаміку

Пріоритет після TASK 05 (цикл має існувати, щоб його перевіряти). САНІТАРНА перевірка, НЕ
головна метрика проєкту (інквізиція 2026-06-23). Прогнати раз, зафіксувати, не перевідкривати.
Читати спершу OVERVIEW.md (критерій успіху) і вихід TASK 05.

## Дві перевірки gate

### Gate A — trust-динаміка змістовна
Чи зсуває розрив обіцяне↔пережите trust у правильний бік, і чи trust зсуває утримання.
- Перевірка 1: на стратегіях, де крива після входу йде ВНИЗ (розрив із обіцяним профілем),
  середній trust персон падає; де крива йде ВГОРУ — тримається/росте. Напрям правильний.
- Перевірка 2: низький trust корелює з виходом/перемиканням, високий — з утриманням.
  (Кореляція trust → дія утримання, знак очікуваний негативний для виходу.)
- Поріг задати ДО прогону: напр. напрям trust правильний у ≥80% стратегій-кейсів;
  |кореляція trust↔утримання| ≥ 0.3. Зафіксувати в reports до результату.

### Gate B — різні причини розрізнювані за спостережуваним
Чи відрізняються точки з різною прихованою причиною за СПОСТЕРЕЖУВАНИМИ полями (окремий
випадок — три мовчання розходяться). Це перевірка достатності полів.
- Фічі: ТІЛЬКИ спостережувані (стан стратегії, режим, час утримання, лог дій, trust, action).
  Прихований вектор і groundtruth — НЕ фічі (assert).
- Метрика: ARI між кластерами і persona_id (та типом мовчання на silence-підмножині). Поріг ARI≥0.5.

## Gate-перевірки перед звітом (assert у коді)
- Жоден рядок без стану стратегії і режиму.
- Прихований вектор НЕ серед фіч.
- groundtruth НЕ серед фіч рішення (він лише для звірки правильності дії, не для кластеризації станів).
- Пороги (A і B) задані ДО результату, не рухаються після.

## Кроки
1. Прочитати датасет TASK 05.
2. Gate A: порахувати напрям trust на down/up-кривих; кореляцію trust↔утримання. Проти порога.
3. Gate B: кластеризація по спостережуваному (KMeans/HDBSCAN); ARI проти persona_id і типу мовчання.
4. Звіт reports/cycle_gate.md: пройшов A? пройшов B? числа, висновок (цикл змістовний + поля достатні
   / що добрати). Без шліфування — пройшов, рухаємось далі.
