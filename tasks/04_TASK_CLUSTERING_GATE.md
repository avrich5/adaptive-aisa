# TASK 04 — Кластеризація як gate інструмента (лінія 2)

Пріоритет 2. Це САНІТАРНА перевірка генератора, НЕ головна метрика проєкту (див. інквізицію 2026-06-23 у CLAUDE.md). Прогнати раз, зафіксувати результат, не перевідкривати.

## Мета gate
Чи відрізняються точки з різною прихованою причиною за СПОСТЕРЕЖУВАНИМИ полями. Окремий випадок: три мовчання (anxious/blind/passive) розходяться у різні кластери. Достатньо -> поля придатні для переходу на живі дані. Схлопуються -> полів мало (корисно знати ДО живих даних).

## Дані
- Вихід харнесу: `data/runs/*.parquet`. Поточний smoke = 5 вікон, 430 рядків.
- Колонки точки:
  - **Спостережувані (тільки ЦІ йдуть у кластеризацію):** `drawdown`, `regime`, `time_in_position`, `recent_action_history`, `action`.
  - **Прихований вектор (НЕ подавати у фічі — це вытік, порушення Gate):** `anxiety`, `trust`, `market_understanding`, `regime_awareness`, `exit_propensity`.
  - **Службові:** `_reasoning`, `_date`, `_close` — не фічі.
  - **Мітки:** `persona_id` — істина для перевірки якості кластерів, НЕ фіча.

## Передумова прогону
- Повний датасет: прогнати харнес через ВСІ 25 drawdown-вікон (не 5). Спершу перерахувати вікна на свіжих даних: `analysis/drawdown_windows.py` (all_regimes оновлено до 2026-06-23). BTC underwater-вікно змінилося — це очікувано.
- seed зафіксувати (42), щоб датасет відтворювався.

## Поріг різнюваності — задати ДО прогону (Gate)
- Метрика збігу кластерів з прихованою причиною: **ARI (adjusted Rand index)** між предсказаними кластерами і `persona_id` (або типом мовчання для підмножини silence-точок).
- Поріг: **ARI >= 0.5** на silence-підмножині = три мовчання розрізнювані. Нижче -> схлопнулись.
- Поріг фіксується в reports ДО подивитися на результат. Не рухати після.

## Кроки
1. Перерахувати drawdown-вікна на свіжих даних (`analysis/drawdown_windows.py`), звірити кількість.
2. Повний прогін харнесу через всі вікна (seed=42) -> датасет у `data/runs/`.
3. `analysis/clustering.py`: фічі ТІЛЬКИ спостережувані; закодувати `regime` (категорія), `recent_action_history` (агрегат), `action`; кластеризація (KMeans/HDBSCAN, k звірити з числом персон/типів).
4. Порахувати ARI проти `persona_id` і проти типу мовчання на silence-підмножині.
5. Gate-перевірки перед звітом: жоден рядок без regime; прихований вектор НЕ серед фіч (assert); поріг заданий до результату.
6. Звіт `reports/clustering_gate.md`: пройшов поріг чи ні, числа, висновок (поля достатні / добрати поля).
