# Synthetic Trader Harness — порядок виконання (залежності)

**Статус:** аналітичний план. Не реалізація.
**Рамка:** прихований "психологічний" вектор персони живе ТІЛЬКИ всередині харнесу,
назовні не виходить і на людях не валідується. Валідується лише спостережуваний
патерн `контекст -> дія`, де контекст складається з полів, наявних і в харнесі, і на
проді (drawdown зі снапшота, режим Base-States, час у позиції, лог дій). Харнес —
генератор гіпотез і тренажер політики, не предиктор.

---

## Крок 1. Консенсус-таксономія станів (handoff Claude Code)

Звести 47 станів від 5 експертів -> 8-12 канонічних User States.
Передумова для всього іншого. Специфікація вже існує: `CLAUDE_CODE_TASK_CONSENSUS.md` (pending).

При зведенні ДОДАТИ те, чого бракує в поточних 47 станах:
- вісь "комунікативність" (пише в чат vs мовчить-спостерігає);
- `silence` як повноправний клас дії;
- біржові дії без чату (`change_capital`, `deactivate_strategy`, `pause_strategy`, `add_strategy`)
  з `wb_strategies.status` + `portfolio_action_log.action`.

Поточні стани описують ПОЧОМУ звернувся, майже не описують ЩО робить між зверненнями.
Мовчун-спостерігач присутній лише маргінально (`inactive_observer` gpt-5,
`whale_watcher`/`whale_monitor` gemini/grok) — винести в перший клас.

Розблоковує: кроки 2-5.

## Крок 2. Специфікація харнесу (аналітичний документ)

Документ у цій папці. Design-constraints:

- **7 персон** (розрахунок у спеці; не 47, не "по одній на стан"; персона != стан).
  Осі: ризик-профіль (low/medium/risky) x розуміння ринку (стежить за режимами vs ні)
  x комунікативність (пише vs мовчить). Повний куб надлишковий -> 7 відібраних,
  з них 3 мовчуни (тривожний / сліпий держатель / пасивний задоволений) —
  ядро гіпотези "мовчання полісемантичне".
- **Тижневі чекпоінти** (12, через drawdown-вікно). Base-States daily granularity ->
  максимум 1 market belief update на добу.
- **Розширений action space** (чат + біржа + silence), тільки з реально логованого.
- **SSR-маппінг** тексту міркування персони на осі прихованого вектора:
  НЕ Лайкерт 1-5, а проєкція на якоря-твердження по кожній осі, усереднення по >=3 наборах.
- **Двохрівнева валідація** (рівень A — внутрішня, відтворюваність маппінгу, можна зараз;
  рівень B — зовнішня, тільки коли є люди, валідується патерн а не вектор).

Прихований вектор (всередині персони): `anxiety`, `trust`, `market_understanding`,
`regime_awareness`, `exit_propensity`.

Це був pending spec з попередньої сесії.

Залежить від: крок 1. Розблоковує: крок 3.

## Крок 3. Реалізація харнесу (implementation session, Claude Code)

- live mode -> signal-emulator;
- replay -> існуючий backtesting (time-travel не потрібен);
- джерело режиму ринку -> Base-States (`/api/transition`, live snapshot, daily);
- ринок реальний (історичні режими, реальна просадка), синтетичний лише трейдер.
- жодного моку.

Цикл одного прогону: персона + початковий набір стратегій з реальної бібліотеки ->
по тижневих чекпоінтах -> зібрати спостережуваний контекст (PnL-снапшот, режим, час у
позиції, історія дій) -> персона ПОРОДЖУЄ текстове міркування (внутрішній монолог,
не повідомлення в чат) -> витягти прихований вектор + дію з action space ->
записати рядок траєкторії `(чекпоінт, контекст, прихований_вектор, дія)`.

Залежить від: крок 2.

## Крок 4. Прогони + виявлення патернів

Накопичити таблицю траєкторій `persona x checkpoint x context x hidden_vector x action`.

Критичний стан = точка траєкторії, після якої різко зростає ймовірність незворотної дії
(`deactivate` / `leave_chat` без повернення). Виявлення:
1. знайти чекпоінти де `exit_propensity` перетинає поріг АБО `silence` повторюється
   >=k разів при зростаючому drawdown;
2. зафіксувати спостережуваний контекст (без вектора): режим, просадка, тижнів мовчання;
3. сформулювати гіпотезу-патерн "контекст C -> критичний стан -> дія A з імовірністю p".

Головний очікуваний патерн (центральний інсайт демо): silent churn — мовчун у просадці
через зміну режиму деактивує, не написавши жодного разу. Харнес має відтворити і дати
числову форму.

Залежить від: крок 3.

## Крок 5. A/B інтервенцій -> демо для Dmitriy

Для кожного критичного патерну — кандидат-інтервенція, тест A/B у харнесі:
- мовчун у просадці + зміна режиму -> проактивне пояснення (адресує низький `market_understanding`);
- тривожний мовчун -> показати що просадка в межах історичного MDD стратегії (адресує `anxiety` фактами);
- сліпий держатель -> пояснити звязок режим<->стратегія до того як деактивує на дні.

Тест: та сама персона через те саме історичне drawdown-вікно двічі — з інтервенцією і без.
Порівняти частку `deactivate`. Це валідація політики до проду (фаза 0 роадмапу `00_CONCEPT.md`).

Інтервенція оперує тим що пояснюване (режим, історичний MDD) -> стикується з verbal-commitment
по explainability перед Vlad. Якщо інтервенція-пояснення знижує churn у симуляції — аргумент
для PRD amendment.

Залежить від: крок 4.
